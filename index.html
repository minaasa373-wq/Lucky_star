DAY1 FRONT / AI超入門

AIは、むずかしい機械じゃない。
まずは「人の作業を助ける賢い道具」だと思えばOK。

AI（人工知能）に苦手意識がある人でも、迷わず読めるように作った。難しい言葉はその場でやさしく言い換え、最後に用語辞典も置いてある。

🤖 AIの正体📚 AIの歴史🧩 生成AI・LLM💬 Gemini

1. AIの正体は？

AIは「人の頭の仕事を少し肩代わりする道具」。電卓が計算を助けるなら、AIは文章作成や情報整理を助ける。

2. AIの歴史は3回ブームがあった

1950年代：ルール型

人が決めたルールで動かす。

1980年代：専門家型

専門家の判断を詰め込む。

今：データ学習型

AIがデータからコツを覚える。

Premiere Proは、最初から全部覚えなくていい。
「片付け → 並べる → 切る → 整える → 出す」でいい。

Premiere Pro（動画編集ソフト）はボタンが多くて怖い。でも初心者が最初に覚える流れは、素材整理、キャンバス作成、カット、文字と音の調整、書き出しだけ。

アニメーション

4. 音声・BGM・SE｜耳が疲れない動画にする

SCENE 03 / 音量は料理の味付け
泉こなた言い換え役

BGMを爆音にすればテンション上がるんじゃない？

柊かがみ解説役

声が聞こえなかったら終わり。目安は声-6dB、BGM-25〜-30dB、SE-12〜-18dBくらい。

dB（デシベル）＝音の大きさの目安。0に近いほど大きい。
BGM

動画全体の雰囲気作り。リミックスツールで動画尺に合わせやすい。

SE

動きやテロップを強調する効果音。入れすぎるとうるさい。

納品方法

初心者の信用

派手な技より、ミスが少ない・納期を守る・確認が丁寧。この3つが強い。

Premiere Pro

業界標準。YouTube、CM、企業PV、テレビ制作など案件に強い。Adobe連携も強い。弱点は高額サブスクと重さ。

3. YAMLプロンプト｜指示を棚に分ける

YAML（ヤムル：データを整理して書く形式）は、AIへの指示を「人物」「背景」「服装」「構図」などに分けるメモ。文章がぐちゃぐちゃになるのを防ぎやすい。

YAMLが向く場面
  • 人物と犬など、複数の対象を分けたい
  • 服装だけ変えたい
  • 同じキャラクター設定を保存したい
  • 世界観をテンプレ化したい
注意点
  • 階層を深くしすぎない
  • 不要な項目を増やしすぎない
  • 雰囲気や感情は自由文で補足する
  • 成功した設定だけ再利用する
最適解：
基本設定はYAMLで整理。空気感や感情は普通の文章で補足。この「構造化＋自由表現」が扱いやすい。

直したい場所を囲む。対象を視覚的に伝えられる。

文字で指示

「中央の女性を男性に変更」「文字を修正」など具体的に書く。

1箇所ずつ修正

複数を一気に直すより、1つずつ直す方が安定しやすい。

履歴に注意

同じチャット内で続ける。別チャットでは続き修正が難しい場合がある。

保存する

完成したらフルサイズで保存。教材やサムネ素材として使う。

限界もある

大量修正や高精度な加工はPhotoshopなど専門ツールも選択肢。

2. インサート素材 4分類｜何のために入れるのかを先に決める

インサート（動画内に差し込む補助映像）は、ただ画面を切り替えるための飾りじゃない。役割を決めて入れると、視聴者が内容をつかみやすくなり、飽きにくくなる。

① 状況説明・理解補助系

難しい話や抽象的な話を、瞬時に分かる絵へ変える。
例：資産形成 → コイン、チャート / 縄文時代 → 生活風景 / 限界挑戦 → 大盛り料理に挑む人。

② 感情リアクション系

驚き、後悔、不安、喜びを視覚化する。
例：悪いニュース → 頭を抱える人 / 失敗 → 落ち込む表情 / 成功 → ガッツポーズ。

③ 抽象概念・イメージ補完系

目に見えない概念を絵に変える。
例：成長 → 上昇グラフ / 分析 → モニターを見る人 / 競争 → 成功した店と衰退した店の比較。

④ データ・情報強調系

数字、比較、制度、仕組みを図解で伝える。
例：100チャンネル運用 → 大きい数字 / 年金制度 → 図解 / 実績 → 比較表。

初心者の注意：
「内容を完全再現する画像」を探しすぎると沼に入る。必要なのは再現度より、視聴者が一瞬で意味をつかめるかどうか。

ステップ4｜番号管理

ステップ5｜一括配置

Premiereでインサート用トラック以外をロックし、マーカー順にまとめて入れる。

ステップ6｜最後に調整

事故防止：

3. 著作権対策｜作品名ではなく特徴を書く

動画生成AIで一番怖いのは、知らないうちに既存作品や実在人物へ寄せすぎること。商用利用を考えるなら、既存作品名やキャラクター名を直接入れるのは避け、特徴へ分解して書く。

危ない書き方

「セーラームーン風」「有名アニメの主人公っぽく」「実在俳優の顔と声で」など、既存作品や実在人物を直接指定する。

安全寄りの書き方

「オリジナルキャラクター」「既存キャラクターへの参照なし」「月明かり、星モチーフ、大きなリボン、青白い光」など、特徴へ言い換える。

重要：
AIが作ったから安全、ではない。最終的に公開・販売する人間が、利用規約、権利、類似性を確認する必要がある。

4. 動画生成プロンプト｜絵だけでなく「動き」と「感情」を書く

画像生成では「何を描くか」が中心だった。動画生成ではさらに「どう動くか」「どんな感情か」「カメラがどう動くか」が重要になる。

弱い例

「かっこいいドラゴンの動画」
→ 動き、カメラ、空気感、火の出方が曖昧でブレやすい。

強い例

「巨大なオリジナルドラゴンが荒野でゆっくり翼を広げ、低く息を吐き、口元から赤い火花が漏れる。カメラは横からゆっくり追走。背景は固定。手足の動きは少なめ。映画風の重い空気。」

動画生成では、動きを増やすほど破綻しやすくなります。まずは顔、髪、煙、光のような小さい動きから始めると安定します。

動きの制御

カメラワーク

固定、ゆっくりズーム、横追走、低い視点、俯瞰などを用途に合わせる。

5. サイズ・手・カメラの事故を減らす

動画生成AIは、手、指、激しい動き、遠近感、口元で破綻しやすい。修正したい時は、対象と変化量と固定要素をはっきり書く。

犬だけ大きくしたい例

「犬のサイズのみを元の5倍に拡大。人物、背景、カメラ角度、光、影は変更しない。犬と床の接地、奥行き、遠近感を維持。」

手の破綻を減らす例

「手は腰の横で固定。指の本数は自然な5本。頻繁な手振りはしない。顔と口元だけ小さく動く。」

2. AIカット自動化｜短尺は人力、長尺はAIが強い

無音カットはAIに任せやすい作業。ManusのようなAIで動画を解析し、Premiere Proで読めるXML（編集データ）として出せれば、大まかなカット作業を自動化できる。

AIに任せる作業

無音検出、言い直し候補の整理、大まかなカット、Premiere用XMLの出力。

人間がやる作業

テンポ調整、発話の頭と語尾の確認、ボケ前の間、会話の空気感、見やすい緩急の調整。

短い素材

1分半くらいなら人力の方が速い場合もある。

長い素材

10分、20分、30分になるとAIの楽さが強くなる。

プロンプト調整

無音判定の長さ、前後の余白、短い発話の扱いを指定する。

大事：
「こんにちは」が「んにちは」になるような事故を避けるため、発話前後に少し余白を残す。AIで大まかに切り、人間がテンポを仕上げるのが現実的。

3. DomoAI｜アニメ・アバター・リップシンク・量産に強い

DomoAIは、動画生成AIの中でもアバター動画、表情、感情表現、リップシンク、アニメ風表現、量産に向いている。テキストだけで作るより、先に画像を作ってから動画化する方が安定しやすい。

画像から動画

実用の中心。原型画像を作ってから動かすと、キャラや構図が安定しやすい。

キャラ動画

参考動画の動きを、原型画像のキャラクターへ反映できる。

リップシンク

音声と口の動きを合わせる。セリフに合わせた表情変化も作れる。

MV制作

画像生成AIで原型、Sunoで曲、DomoAIで歌唱カットや振り付けを作る。

AIアバター

VTuber風画像やキャラ画像に音声を合わせて喋らせる。

動く漫画・アニメCM

漫画風画像や広告構成を動画化し、ナレーションとBGMを合わせる。

使い方の基本：
DomoAIは「先に良い画像を作る → 動画化 → リップシンクや動きを追加 → 編集ソフトで仕上げる」の順番が分かりやすい。
